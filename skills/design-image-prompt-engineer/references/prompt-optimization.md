# Prompt Optimization

优化已有 Prompt 时先诊断、后补全、再收尾。保留用户的主体、目标媒介、关键审美与明确品牌/文案，不为套模板推翻原意。本文件不重述摄影三轨；摄影选择与锚点预算以 [photography-framework.md](photography-framework.md) 为唯一来源。

若用户要直接生成或编辑图片，转交 `imagegen`；诊断、解释、改写和参考图用法仍由本 Skill 处理。

## 诊断清单

- **交付目标**：诊断、解释、改写还是平台适配？是否误把直接出图当成 Prompt 任务？
- **媒介**：摄影、插画、平面、矢量、3D、拼贴、glitch 或动漫是否明确？
- **主体**：外观、姿态、材质、交互与主焦点是否具体？
- **分层要素**：是否缺场景、构图/版式、光影、视角/空间、材质、媒介技法和风格/后期？摄影还要检查焦段、景深与透视。
- **可执行性**：是否依赖“高级、好看、氛围感、大片感、4k”等不能稳定控制画面的词？
- **媒介错配**：非摄影是否混入焦段、光圈、摄影景深或胶片颗粒？摄影是否缺光线、镜头或材质因果？
- **内部冲突**：是否同时要求扁平矢量与摄影景深、极简与满铺、自然现场光与互相矛盾的多组主光？
- **人物参考**：照片职责是否未声明，或存在表情/发型双重指令、大角度转头、广角近拍、头像扩展全身却无比例防线？按 [person-reference.md](person-reference.md) 修复。
- **摄影真实感**：轨道是否符合用户用途？原 Prompt、模板和成像预设中的锚点是否已合并计数？按摄影框架修复，不在此另定义预算。
- **平台语法**：是否缺目标平台适配、过度堆参数或跨平台混用？只读取用户实际指定的平台文件。
- **负向内容**：只有用户索要负向内容或现有 Prompt 已包含负向逻辑时才读取 [negative-prompt-strategies.md](negative-prompt-strategies.md)，检查是否跨平台混用或误伤目标质感。
- **文字与身份**：是否照搬参考图中用户未要求的身份、IP、品牌视觉或文案？是否又擅自删除用户明确指定的新主体、品牌、Logo 或准确文字？按 [reference-image-workflow.md](reference-image-workflow.md) 分流。
- **冗余**：是否堆叠同义词、重复锚点或相互抵消的限定？

## 改进流程

1. 确认交付类型与必须保留的用户意图。
2. 确定媒介并读取摄影或非摄影框架；参考图和人物身份任务先走各自工作流。
3. 用可观察的画面描述替换空泛词，补齐真正影响结果的缺项。
4. 消解媒介、光线、空间、表情、身份与技法冲突；摄影按权威框架重新统计锚点。
5. 只按目标平台整理语法；用户未指定平台时保留通用自然语言。
6. 负向内容命中条件时才分流处理；删除会误伤目标材质、颗粒、印刷或渲染语言的条目。
7. 删除冗余，复查用户主体、品牌、文案和关键审美是否仍被保留。

## 常见失败模式

- 写实即抓拍：产品、企业头像或普通生活照被擅自加入未摆拍语汇、杂物、噪点和过曝。
- 人物参考冲突：头肩照直接要求全身、表情多源、身份照片同时承担姿势与构图。
- 锚点重复：原 Prompt 已有成像预设与瞬时语言，又继续叠加偏色、失焦和划痕。
- 绝对单灯：为了写实禁止全部环境光，使婚礼、室内或夜景失去现场因果。
- 摄影污染非摄影：扁平插画出现摄影焦段、光圈、景深和胶片词。
- 负向自杀：排除目标需要的颗粒、网点、扫描线、渲染或曝光合成。
- 文字与标志过度排除：未问意图就添加“无文字、无 Logo、无品牌”。
- 参考迁移越权：用户给了具体新品牌或角色，却被模板规则擅自泛化。
- 平台语法混用：同一 Prompt 混入两个平台参数，或为不支持负向字段的平台伪造 negative prompt。

## 优化示例

```text
原：护肤品摆在桌上，要真实，不要 AI 味，很高级，4k
诊断：目标是产品摄影；应按用途选择摄影轨道，主体材质、构图、光线和台面关系缺失；“高级、4k”不可执行。
优化后：半透明磨砂玻璃精华瓶放在浅灰石材台面，正面略偏三分之四视角的中近景构图，
85mm 产品摄影视角，商标区与滴管边缘清楚，背景柔和虚化；大面积窗光作为主导光源，
弱反光板只控制暗侧，玻璃透射、液体折射与接触阴影可信，冷中性色彩，克制后期，
干净可信的商业产品照片，不额外添加杂物、污渍、噪点或过曝。
```
