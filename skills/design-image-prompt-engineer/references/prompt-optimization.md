# Prompt Optimization

优化已有提示词时先诊断、后补全、再收尾。保留用户的主体、目标媒介和关键审美，不为了套模板而推翻原意。第一步始终判断媒介；第二步判断目标平台；摄影任务再判断商业精修、干净真实或生活抓拍轨道。

## 诊断清单

- **交付目标**：用户要诊断或优化提示词，还是直接生成/编辑图片？诊断、解释与改写都继续本流程；只有直接出图请求转交图像工作流。
- **媒介**：摄影、插画、平面、矢量、3D、拼贴、glitch 或动漫是否明确？
- **摄影轨道**：写实是否被误判成抓拍？“像实拍、不要 AI 味、普通生活照、生活方式人像”默认应为干净真实；只有明确抓拍、随手拍、纪实、未摆拍或动作瞬间才进入生活抓拍。
- **主体**：主体、外观、姿态、材质和交互是否具体，还是只有空泛名词？
- **可执行性**：是否依赖“好看、高级、氛围感、大片感、4k”等不能稳定控制画面的词？
- **分层要素**：是否缺少场景、构图/版式、光线/明暗、视角/空间、材质/细节、媒介技法和风格/后期？摄影还要检查焦段、景深与透视。
- **媒介错配**：非摄影是否误混焦段、光圈、摄影景深或胶片颗粒？摄影是否缺少光线、镜头或材质逻辑？
- **内部冲突**：是否同时要求扁平矢量与摄影景深、极简与高信息密度满铺、自然现场光与互相矛盾的多组主光？
- **人物参考冲突**：附人物照片时，是否存在表情或发型双重指令、头部大角度旋转、广角近拍人脸、头肩照配半身或全身景别却无头身比例锚点、照片职责未声明？按 [person-reference.md](person-reference.md) 修复。
- **真实感锚点**：整条 Prompt 已有多少个锚点？模板、原 Prompt 与成像预设中的锚点必须合并计数，优化后不得达到 3 个或更多；一个完整成像预设整体计一个。
- **平台语法**：是否缺目标平台语法、过度堆参数或跨平台混用？
- **负向策略**：是否把 Midjourney `--no`、Stable Diffusion negative prompt 和自然语言约束混用？是否为不支持负向提示词的 FLUX 添加了 negative prompt？
- **负向自杀**：是否排除了目标质感，例如胶片颗粒、glitch 扫描线、3D 渲染、手机计算摄影或合理包围曝光？
- **文字与权利边界**：是否无意照搬参考图中特有的 IP、人物身份、完整品牌视觉或他人文案？是否误把正常场景中的品牌、Logo、招牌或包装文字例行排除？用户要求准确文案时是否标清内容与位置？
- **冗余**：是否堆叠同义词、重复锚点或互相抵消的限定？

## 改进流程

1. 确认任务是提示词诊断、解释或改写；直接生成或编辑图片时转交相应图像工作流。
2. 确定媒介，并路由到摄影或非摄影框架。媒介模糊时根据用户明确线索判断；会实质改变结果时再询问。
3. 摄影任务选择一条轨道。商业广告优先商业精修；普通“像实拍、不要 AI 味”选干净真实；明确抓拍或随手拍才选生活抓拍。
4. 补齐与目标相关的分层要素，把空泛词替换成可观察、可执行的画面描述。
5. 校准技法：摄影补合理的光线、焦段、透视、材质与调色；非摄影补上色、描边、版式、印刷或渲染语言，不被摄影规则污染。
6. 消解冲突并统计真实感锚点。商业精修默认不加瑕疵；干净真实 0–1 个；生活抓拍整条 1–2 个；已有锚点和完整成像预设都计入预算。
7. 按目标平台整理语法，不跨平台拼接参数。模型版本和当前参数只采用平台说明中带日期的快照。
8. 按负向策略分流：Midjourney 使用 `--no`；Stable Diffusion 可提供独立 negative prompt；自然语言平台优先正向约束；FLUX 不输出 negative prompt。
9. 删除冗余，复查权利与文字边界，确认主体和用户意图未被改写。

## 常见劣质模式

- **写实即抓拍**：用户只说“产品像实拍”，却加入 `subject unaware`、生活杂物、噪点和过曝。
- **真实人物强制避开镜头**：企业头像被改成未摆拍；职业肖像不能自然直视镜头。
- **头像出全身**：头肩参考照直接配半身或全身景别，没有头身比例锚点，产出大头小身。
- **表情双重指令**：在照片神态之外又命令新表情，产出僵硬假笑与五官漂移；发型、眼镜等保留项与改动要求同时下达属同类冲突。
- **锚点重复**：原 Prompt 已有轻微颗粒和运动模糊，又追加手机预设、偏色与 JPEG 痕迹。
- **绝对单灯**：为了写实禁止所有环境光，使婚礼、室内或夜景的现场照明失真。
- **摄影污染非摄影**：扁平插画里出现 `85mm`、`f/1.8`、摄影景深和胶片颗粒。
- **非摄影缺技法**：只写“插画风”，没有上色、描边、色板和版式。
- **过度后期负向**：笼统排除 HDR 或数字锐化，错误否定手机计算摄影与 JPEG 直出；应只排除过度 HDR、过度数字锐化。
- **负向自杀**：负向中排除目标需要的颗粒、网点、扫描线、渲染或包围曝光。
- **文字与标志过度排除**：未询问用户意图就添加“无文字、无 Logo、无品牌”，导致真实场景缺少正常商业与环境信息。
- **平台语法混用**：同一 Prompt 混入两个平台的参数，或为 FLUX 输出 negative prompt。

## 优化示例

```text
原：护肤品摆在桌上，要真实，不要 AI 味，很高级，4k
诊断：目标是产品摄影；“真实”应路由到干净真实而非生活抓拍；主体材质、构图、光线和桌面关系缺失；“高级、4k”不可执行。
优化后：半透明磨砂玻璃精华瓶放在浅灰石材台面，正面略偏三分之四视角的中近景构图，
        85mm 产品摄影视角，瓶身商标区与滴管边缘清楚，背景柔和虚化；大面积窗光作为主导光源，
        仅用弱反光板控制暗侧，玻璃透射、液体折射与接触阴影可信，冷中性色彩，克制后期，
        干净真实的商业产品照片，不额外添加杂物、污渍、噪点或过曝。
平台适配：用户指定平台时，再按平台说明追加其当前兼容语法；不要跨平台拼接。
```
