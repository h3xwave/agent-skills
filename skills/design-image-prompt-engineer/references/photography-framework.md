# Photography Framework

本文件是摄影三轨、真实感锚点和主导光源规则的唯一权威来源。其他摄影词库只提供可选语言，不得重新定义轨道或预算。

## 组装流程

1. 按用途选择且只选择一条摄影轨道。
2. 写清主体、场景、画面用途、构图、镜头、主导光源、材质和后期。
3. 用户明确指定手机、胶片、一次性相机、早期 CCD、扫描底片等成像链路时，最多叠加一个成像修饰器。
4. 统计用户 brief、模板、成像修饰器和补充词中的全部真实感锚点，达到预算后停止添加。
5. 检查人物视线、光源因果、曝光、透视和后期是否符合所选轨道。

## 摄影三轨

| 轨道 | 进入条件 | 锚点预算 | 处理原则 |
| --- | --- | ---: | --- |
| 商业精修 | 广告、品牌 campaign、电商主图、杂志美妆、奢侈品 hero shot、精确棚拍 | 默认 0 | 可用受控多灯、精确反射、焦点堆叠、干净材质和精细但自然的后期；不为“真实”主动加瑕疵 |
| 干净真实 | 像实拍、照片感、不要 AI 味、自然皮肤，或只要求可信照片 | 0–1 | 合理镜头、可信材质、自然曝光与阴影、克制后期；轻微锚点可选而非必需 |
| 生活抓拍 | 明确要求抓拍、随手拍、纪实、未摆拍、派对快照、偷拍视角或动作瞬间 | 1–2 | 现场光或直接闪光优先，保留合理瞬时感；只按意图使用 `subject unaware`、`mid-motion` 或偶然构图 |

普通生活照、生活方式人像、企业头像、证件式肖像、产品、建筑、房产、汽车和风光即使要求“真实”，也先走干净真实。企业头像、正式人物照和合影可以自然直视镜头；不要把“真实”自动解释为 candid、低质或脏乱。

## 真实感锚点预算

锚点是为打破合成感而主动加入的成像签名、现场微瑕、生活痕迹或瞬时语言。合理焦段、可信透视、主体材质、正常曝光和自然阴影属于基础摄影信息，不算锚点。

- 按整条 Prompt 计数：用户要求、原 Prompt、模板和成像修饰器全部纳入同一预算。
- `subject unaware`、`mid-motion`、轻微手震、背景生活痕迹等独立短语各算一个。
- 一个完整成像预设整体算一个，即使内部包含相互配套的颗粒、偏色或 JPEG 特征。
- 用户明确指定的特征优先占用预算；达到预算后停止添加，不同时堆噪点、过曝、杂乱、倾斜、失焦和划痕。
- 用户指定胶片或手机等预设时，可覆盖商业精修默认 0 个锚点；计为 1 个后不再主动加其他瑕疵。

示例：手机婚礼抓拍可用“手机成像预设（1）+ `mid-motion`（1）”；护肤品自然反射与接触阴影是基础信息，可保持 0 个锚点。

## 成像修饰器

成像修饰器不改变摄影轨道；通常只选一个，并整体计一个锚点。

- 手机直出：轻广角、计算摄影曝光、适度机内锐化、自然 JPEG 质感；不同时要求低动态范围和全面高光溢出。
- 35mm 胶片：自然颗粒、柔和高光过渡、轻微色彩响应；按指定胶片与场景调整，不自动加划痕。
- 一次性相机：固定焦段、直接闪光、较粗颗粒、有限控制；适合旅行、聚会和青春记录。
- 早期 CCD：消费级 JPEG 色彩、有限分辨率，可按场景加入直闪或合理高光截断。
- 扫描底片：胶片色罩、扫描颗粒，可有轻微尘点或边缘痕迹；不堆叠所有老化损伤。
- 一次成像：轻微软化、化学显影色偏、相纸边框；适合私人记忆感。

需要更具体的胶片与数字调色时读取 [color-grade-library.md](color-grade-library.md)。

## 主体与环境

- 说明主体类型、外观或材质、姿态与动作、交互、主焦点和辅助元素。
- 视线服从用途；不要为了真实感强迫人物避开镜头。
- 场景写清时间、天气、背景层次、空间尺度和有因果的道具。
- 干净真实不等于杂乱；产品、建筑、房产、汽车默认不加垃圾、灰尘、指纹、破损或无关路人。
- 建筑与房产保持空间比例、垂直线、窗内外亮度和陈设秩序；允许合理包围曝光与克制合成。

## 构图、镜头与动作

- 按用途选择一个主结构，如中心、三分、对称、对角线、框中框、负空间或前景层次；需要详细词汇时读取 [composition-camera-language.md](composition-camera-language.md)。
- 写明机位、景别和画幅。商业精修与干净真实可以精确构图；只有明确抓拍时才用偏轴、不完美裁切或偶然遮挡。
- 人像常用 50mm–85mm；环境人像可用 35mm。焦段和拍摄距离优先于盲目追求极浅景深。
- 产品常用 50mm–100mm 标准或微距；按用途选择单点清晰或克制焦点堆叠。
- 建筑与房产用不过度拉伸的合理广角并校正透视；汽车用标准至中长焦控制车身比例。
- 运动与纪实只有在动作和快门逻辑需要时才加入运动模糊。

## 可解释的主导光源

干净真实与生活抓拍要有一个决定曝光与主要阴影的主导光源，例如窗光、阴天漫射、顶灯、路灯、直接闪光或摄影主灯。允许环境光、实景灯具和弱辅助光，只要不争夺主导关系。

- 婚礼、餐厅和街景等混合光环境先说明谁主导曝光，再保留现场色温差。
- 干净真实可用弱反光板或低功率补光控制反差，阴影方向仍需可信。
- 生活抓拍优先现场光或直接闪光，不堆满主光、补光、轮廓光和发丝光。
- 商业精修允许受控多灯与复合反射，但每盏灯必须服务材质、脸型、轮廓或背景。
- 需要具体布光结构时读取 [lighting-techniques.md](lighting-techniques.md)。

## 后期与 AI 味诊断

- 塑料皮肤或过度磨皮：改为自然肤质与克制修饰，不自动加入痘印、油光和杂毛全套。
- 过度 HDR：恢复自然明暗层级与克制色调映射；房产可合理合成但避免边缘光晕和局部反差均质化。
- 过度数字锐化：使用符合镜头、设备和观看尺寸的细节频率；正常手机计算摄影不需排除。
- 图库式姿态：先判断用途；正式肖像可看镜头，只有明确抓拍才用瞬时语言。
- 材质与反射不可信：写清粗糙度、反射形状和接触阴影，而不是加灰尘或磨损。
- 画面均匀发亮：重新确定主导光源、方向、衰减和主要阴影。

## 自检

- 是否只选择一条轨道，且锚点总数未超预算？
- 成像修饰器是否最多一个并按整体计数？
- 主导光源、环境光、曝光和阴影是否因果一致？
- 人物视线与构图是否服务用途，而非机械套用抓拍？
- 产品与空间是否保持干净可信，没有把低质量误当真实？
