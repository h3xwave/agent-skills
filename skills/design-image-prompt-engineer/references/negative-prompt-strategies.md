# Negative Prompt Strategies

仅当用户明确索要 negative prompt、`--no`、排除项，或现有 Prompt 已包含负向逻辑时读取本文件。先确认平台，再决定是否输出负向内容；禁止把 Midjourney `--no`、Stable Diffusion 独立 negative prompt、自然语言平台的正向约束共享成一套“通用负向词”。FLUX 不支持负向提示词，不为 FLUX 输出 negative prompt。

## 平台分流

| 平台类型 | 输出方式 | 规则 |
| --- | --- | --- |
| Midjourney | 使用独立的 `--no item1, item2` 参数 | 只列确实要排除的简短对象或属性；不要把 Stable Diffusion 的长串 negative prompt 塞进 `--no`，也不要把 `--no` 混入自然语言正文 |
| Stable Diffusion | 在独立 negative prompt 字段输出 | 与正向 Prompt 分栏；按模型和工作流适度使用，不把目标媒介的必要质感写入负向 |
| GPT Image、Nano Banana 等自然语言平台 | 优先在正向 Prompt 中写清“保持什么”以及可接受边界 | 使用“自然肤质、克制后期、背景简洁”等正向替代表达；除非当前平台界面明确提供独立负向字段，否则不伪造 negative prompt 参数 |
| FLUX / FLUX Kontext | 只输出正向替代表达 | FLUX 不支持负向提示词；把排除意图改写为期望的画面状态，不添加 negative prompt、`--no` 或 Stable Diffusion 语法 |

同一任务只走目标平台对应的一条路径。用户未指定平台时，默认提供正向、自然语言约束；确有必要时可以另给“需避免的语义目标”，但不要冒充某个平台的可粘贴负向字段。

## 排除目标

先写清要保留的目标，再只排除会破坏目标的内容：

- 结构错误：多余肢体、错误解剖、几何穿模、重复物体、错误接触关系。
- 不需要的生成痕迹：与用户意图冲突的乱码、水印、错误或重复 logo、无意的伪影。正常场景中的品牌、Logo、招牌和包装文字不属于默认排除项。
- 风格偏离：只排除与目标媒介相冲突的表现，不排除目标本身依赖的纹理。
- 后期过量：使用“过度 HDR、过度数字锐化、过度磨皮、过度商业修图”，不要笼统排除所有 HDR、锐化、颗粒、过曝或阴影。

不要把 `logo` 或 `text` 无条件列入负向；只有用户明确要求纯图、无文字或无品牌时才排除。用户要求准确呈现特定品牌标志或文字时，描述允许出现的内容与位置，并只排除额外、错误、重复或乱码元素。

## 媒介策略

以下条目是“排除语义候选”，必须再按目标平台转换成上面的输出方式，不能原样复制为跨平台通用负向词。

### 摄影

摄影轨道、真实感锚点与成像修饰器以 [photography-framework.md](photography-framework.md) 为唯一来源；本文件不重新定义三轨。根据已选轨道保留目标所需的现场感与成像特征，只排除会破坏任务的项目：

- 可排除错误解剖、反射穿帮、重复元素、非预期污渍、过度 HDR、过度数字锐化、塑料皮肤和过度磨皮。
- 产品与静物只有在用户要求干净商业画面时才排除杂乱背景，不消除所有真实材质纹理。
- 噪点、轻微过曝、局部暗部、运动模糊、随手构图和混合环境光可能是明确抓拍目标，不得例行排除。
- 胶片或旧设备链路依赖的颗粒、色偏、硬闪阴影、扫描痕迹和 JPEG 痕迹应保留；只排除非预期严重糊片、结构错误或相冲突的数字处理。

### 干净矢量、扁平与 Cel

优先正向写“纯色块、统一描边、边缘清楚、无摄影纹理”。可排除非预期照片写实、3D 体积、脏乱线条、色彩溢出与糊边；若用户明确要做旧矢量或混合媒介，不排除颗粒和套色痕迹。

### 做旧、拼贴、Glitch 与印刷

保留半调网点、扫描线、套色错位、纸张磨损、压缩块和有意噪点。可排除无意的塑料光洁感、不可辨识主体与随机生成乱码，但不要排除构成风格的数字或印刷错误。

### 3D / CGI

优先正向写“几何接触可信、材质响应一致、反射受控、渲染语言统一”。可排除几何穿模、悬浮接触、错误反射、非预期廉价塑料感；不要排除渲染、全局照明、环境遮蔽或用户指定的风格化材质。

## 交付检查

- 是否先按目标平台选了唯一输出方式，而不是复用一套通用负向词？
- Midjourney 是否只使用 `--no`，且没有 Stable Diffusion 的独立字段格式？
- Stable Diffusion 的 negative prompt 是否与正向 Prompt 分开？
- 自然语言平台是否优先使用正向约束？
- FLUX 是否完全没有 negative prompt 或 `--no`？
- 是否保留了胶片颗粒、glitch 扫描线、拼贴套色、3D 渲染等目标质感？
- 摄影负向是否写成“过度 HDR、过度数字锐化”，避免与手机计算摄影、JPEG 直出或克制包围曝光冲突？
