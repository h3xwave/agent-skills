# Person Reference Consistency（人物参考一致性）

用户提供人物照片并要求“同一个人”出现在新画面，或询问人物比例、表情与身份一致性为什么失败时使用本文件；本 Skill 只交付原因诊断、Prompt 或参考图用法说明，直接生成或编辑图片转交 `imagegen`。头像通常只能可靠提供画面中可见的面部与发型信息，未展示的体型、全身比例和姿态不是可保留的事实；生成系统还可能继承照片的裁切、透视、光线或表情。Prompt 用于声明职责并补齐目标，结构稳定性还要依靠合适的取景、额外参考或两步编辑，不能由文字单独保证。

仅处理用户有权使用的人物照片（本人或已获授权者）；名人与公众人物肖像复刻不属于本 Skill 任务。

## 照片职责声明

- 先按可见范围判断照片职责：头像可提供面部身份；半身或全身照可按用户意图提供体型与比例；姿势图提供动作和重心；其他图片分别提供构图、场景或风格。每张参考图只承担明确列出的职责。
- 只有头像时默认写“参考照片中的人物仅提供面部身份”；未展示的身体按用户说明或中性假设生成，不得声称生成体型就是本人真实体型。
- 写明保留项（五官、脸型、发型与标志性特征）与不继承项（照片的构图、裁切、背景与光线），否则模型可能连头像的取景一起继承。
- 眼镜、胡须、痣等附属特征逐项点名去留；用户要求改发型、妆容或摘镜、剃须时，把该项移出保留项、只下一个目标指令（保留句相应改为“保持五官与脸型一致”），并说明强辨识特征改动越大、相似度越低。
- 多图任务逐张声明用途：按实际输入从“面部身份、体型与比例、姿势与重心、构图、场景、风格”中选择，例如图 1 提供面部身份、图 2 提供全身体型与姿势、图 3 提供场景或风格；不要让同一张头像同时承担身份、全身结构和构图，编号必须对应用户实际图序。
- 多人各带参考照时逐人绑定位置与外观：“图 1 人物位于画面左侧、[服装与动作]，图 2 人物位于右侧、[服装与动作]”，并写明“两人面部互不混合”；单次参考能力不足时，改用两步法逐个注入面部。

## 比例防线

- **取景匹配**：默认让输出景别贴近照片取景，头肩照最稳的输出是头肩或胸像构图。
- **全身参考优先**：头像要扩展为半身或全身时，平台支持多参考就优先补一张有权使用的全身或姿势参考，分别锁定体型、重心和四肢关系；没有额外参考时明确身体是生成假设，并优先稳定静态姿势或两步法。
- **头身比例锚点**：头肩照要输出半身或全身时，必须写比例锚点，例如“成年人自然头身比例，头部与肩宽、躯干协调”，并把体态、服装写具体，不留给模型猜。锚点按目标年龄段改写并写明年龄段：儿童按具体年龄描述成长阶段和相应体态，老人写含年龄的体态与颈肩姿态，防止儿童被成人化、老人被年轻化。
- **体态与裁切**：坐、蹲等非站立体态写明支撑面与身体接触点，四肢与躯干比例协调；半身裁切落点选胸口或腰部、不落在肘腕关节；手要入画就写明双手动作与位置，否则让构图避开手部。
- **稳定姿势默认**：用户重点解决比例或表情且未要求动作时，默认放松站立、双脚稳定着地、重心清楚、双手动作简单且互不交叉；不要自行增加走路、转身、跳跃或大幅摆臂。
- **人像镜头**：近景与半身默认 50mm–85mm 平视透视；全身默认 50mm 左右并拉开拍摄距离。不用广角或鱼眼近拍人脸，不用大幅仰拍俯拍，除非用户明确要求并接受面部透视变形。
- **脸部占比**：人物在画面中过小时面部最先崩坏；优先头肩、半身或中景，必须远景或群像时改用两步法。

## 表情防线

- **表情主权唯一**：默认写“表情沿用照片中的自然神态”；用户要求新表情时删除其余表情词，只保留一个目标神态，并说明目标表情与照片差异越大、相似度越低。
- **用神态描述代替表情命令**：写“神态放松、嘴角微松、眉眼自然”，不写“大笑、惊讶、夸张表情”这类与照片打架的硬命令；视线默认沿用照片，企业头像等正式肖像可自然直视镜头。
- **头部角度默认与照片一致**：侧脸、仰头等大角度旋转会降低相似度并造成五官漂移，需用户确认后再写入。
- **面部光线单一可信**：一个可解释的主导光源，不在人脸上叠加与照片剧烈冲突的强戏剧布光或多组彩色光。

## 身份与风格优先级

- 写实任务身份优先：默认进入干净真实轨道，按 [photography-framework.md](photography-framework.md) 组装，真实感锚点预算照常执行。
- 风格化任务（插画、3D、动漫等）风格优先：面部会被媒介风格化，靠发型、脸型轮廓、配饰等标志性特征保持辨识度，不承诺照片级相似；不要输出写实人脸拼接风格化身体的混合体。
- 目标为具名 IP 或工作室画风时，按非摄影风格库的权利边界处理：用通用媒介技法与流派语言定调（如手绘水彩背景、赛璐珞平涂、柔和自然光），不复刻该工作室的签名视觉与角色设计。
- 用户同时要求高相似度与强风格化时，说明二者互相牵制，请其选择优先项，或改用两步法。

## Prompt 骨架

### 头肩或半身

```text
参考照片中的人物仅提供面部身份：保持五官、脸型与发型一致，
表情沿用照片中的自然神态，不继承照片的构图、背景与光线。
[人物动作与姿态]，位于 [场景]，[头肩/半身构图]，平视角度，85mm 人像透视，
[目标年龄段]自然头身比例，头部与肩宽协调，[服装与材质]，
[一个可解释的主导光源与色彩方向]，[选择的摄影轨道收尾]
```

### 头像扩展为全身

```text
图 1 中的人物仅提供面部身份：保持五官、脸型、发型与明确保留的标志性特征；
[可选：图 2 仅提供全身体型、比例与姿势]，不继承各参考图的背景、裁切、光线或无关人物。
[目标年龄段]，[用户指定的体型；若未提供则写明采用中性生成假设，不代表本人真实体型]，
放松稳定站立，双脚自然着地、重心清楚，双手动作简单且互不交叉，
[唯一目标神态]，[服装与材质]，位于 [场景]，普通生活方式人像而非动态抓拍，
竖版全身构图，从头顶到鞋底完整入画，人物占画面主体，平视角度，约 50mm 自然透视，
自然头身比例，头部、肩宽、躯干与四肢协调，[主导光源与色彩方向]，[干净真实收尾]
```

双参考图按实际用途改写；图 2 是风格图时写“仅提供风格与色彩”，是全身或姿势图时写“仅提供体型、比例与姿势”。编号按实际图序对应，人物照在后时相应互换。

## 两步法

远景、复杂场景、群像，或高相似度与强风格化互相牵制时，把任务拆成两步：先生成场景与泛化人物，锁定构图、比例与光线；再在支持参考编辑的图像工作流中注入参考照片的面部身份，明确“只替换面部身份，姿态、构图与光线保持不变”。画面为风格化媒介时，加写“替换后的面部保持该媒介的风格化程度，仅注入身份特征，不回退为写实人脸”。比例由第一步锁定，身份由第二步注入；实际出图步骤由 `imagegen` 或用户所选图像工具执行。

## 照片质量门槛

清晰正面或微侧、面部无遮挡、无美颜滤镜与重度磨皮、单人、面部像素充足。近距离广角自拍的鼻脸透视不作为身份几何继承；优先换用拍摄距离更自然的照片。美颜照片自带非自然几何，生成时会被继续放大；表情与角度贴近目标画面的照片，效果远好于靠 Prompt 硬掰。

## 失败模式速查

| 症状 | 常见原因 | 修复 |
| --- | --- | --- |
| 大头小身、头身失调 | 头肩照直接要求半身或全身，没有比例锚点 | 取景匹配，或写入头身比例锚点并具体描述体态与服装 |
| 表情僵硬、假笑 | 表情双重指令：照片神态之外又命令新表情 | 表情主权唯一，用神态描述代替表情命令 |
| 五官漂移、不像本人 | 大角度旋转、风格化过强或身份约束不足 | 角度回正，明确身份与风格优先级，必要时补充更合适的参考照片 |
| 贴脸感、脸与身体脱节 | 身份约束过强，或面部光线与场景光冲突 | 减少身份参考对姿态和构图的干预，统一主导光源与色温 |
| 面部细节崩坏 | 人物在画面中占比过小 | 改头肩、半身或中景；必须远景时用两步法 |
| 双人合影两张脸互相融合 | 多人参考未逐人绑定，或超出平台单次身份参考上限 | 逐人绑定位置与外观并声明面部互不混合；必要时两步法逐个注入 |
| 年龄漂移：儿童成人化、老人年轻化 | 套用成人比例锚点，未写明目标年龄段 | 按年龄段改写头身比例锚点，并在 Prompt 写明年龄段 |
