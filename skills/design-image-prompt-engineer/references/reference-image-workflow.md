# Reference Image Workflow

用于参考图的风格分析、可替换主体模板和带具体新主体的完整 Prompt。先确认交付模式，再提取可迁移视觉语言；不要让模板契约覆盖用户明确要的分析或具体主体。

若用户要求保留照片中的同一人物，改走 [person-reference.md](person-reference.md)。若用户要求直接生成或编辑图片，转交 `imagegen`；本文件只交付分析、Prompt 或参考图用法。

## 三种模式

| 模式 | 进入条件 | 输出 |
| --- | --- | --- |
| 风格分析 | “分析这张图的风格 / 为什么像这种质感 / 拆解画面” | 用清晰的小标题或短列表说明关键维度、相互作用和可迁移做法；只有用户同时索要 Prompt 时才追加 Prompt |
| 可替换主体模板 | “反推通用 Prompt / 提取风格模板 / 保留风格以后换主体”，且没有给出具体新主体 | 只输出一段可粘贴的模板 Prompt，使用一次主体占位符，不输出分析、负向词或平台参数 |
| 具体主体完整 Prompt | 用户明确给出新人物、产品、品牌、角色、物体或场景，并要求迁移参考风格 | 输出包含该具体主体的完整 Prompt；可按请求附简短迁移说明或平台适配 |

用户意图同时覆盖多个模式时，按其明确要求组合输出并标清各部分；不得默认把“分析”改成纯 Prompt，也不得用占位符替换已经提供的具体主体。

## 分析维度

只选择对当前画面有解释力的维度，不机械逐项罗列：

1. 媒介与总体取向：摄影、插画、平面、像素、3D、拼贴或混合媒介。
2. 主体层次与画面成分：前景、中景、背景及视觉焦点。
3. 构图、景别与观察角度：中心、三分、对称、对角、框中框、俯仰视或轴测等。
4. 光影或插画明暗：方向、软硬、对比、体积与阴影逻辑。
5. 色板与色彩科学：冷暖、饱和、对比、互补关系、印刷油墨或调色特征。
6. 媒介技法与材质：笔触、描边、上色、纸张、网点、渲染器、表面响应。
7. 空间、透视、信息密度与留白。
8. 动态、时代文化语境、情绪与叙事张力。
9. 后期或工艺痕迹：颗粒、漏光、色差、扫描线、压缩块、套色错位等。
10. 一眼可识别的视觉签名，以及它与其他维度如何共同成立。

摄影参考图可按需读取 [lighting-techniques.md](lighting-techniques.md)、[composition-camera-language.md](composition-camera-language.md) 与 [color-grade-library.md](color-grade-library.md)；非摄影参考图可读取 [non-photo-style-library.md](non-photo-style-library.md) 与 [illustration-technique-library.md](illustration-technique-library.md)。

## 迁移与身份边界

- 迁移构图、光影、色彩、材质、媒介、空间、氛围、动态和工艺，不把图像识别、OCR、人物鉴定或品牌确认伪装成风格分析。
- 参考图附带但用户未要求迁移的具体人物身份、IP、商标、品牌、地标、台词、可读文字和专属故事元素应删除或泛化，只保留可复用美学。
- 用户明确指定的新主体、自有角色、品牌、Logo、文案或其他身份元素应原样保留并写入完整 Prompt；不要仅因它具体或具名就擅自泛化。准确文字需写明内容、位置和层级。
- 用户明确要求保留参考图中的某个可用元素时，按其授权与上层安全规则处理；不要假装未收到这一要求。
- 模板模式不写具体主体；具体主体模式不使用模板占位符。

## 可替换模板契约

模板 Prompt 的核心主体位置原样出现且只出现一次：`[在此处替换为您想要生成的主体内容]`

- 直接输出一段完整、连贯的中文 Prompt；不加标题、前言、分析、列表、解释或寒暄。
- 使用一段或最多三段自然语言，不使用 Markdown 代码块。
- 把参考图的可迁移视觉语言压缩进 Prompt，不逐项展示分析维度。
- 不附负向提示词、平台参数或平台使用说明；只有用户明确要求另附时，才在模板之后单独处理。

## 具体主体契约

- Prompt 开头尽早写清用户给出的主体及其必须保留的身份、品牌、材质或文字。
- 让参考图提供风格与构图，不让其原主体覆盖新主体；多图时逐张声明“主体 / 风格 / 构图 / 场景”的职责。
- 按新主体调整尺度、姿态、交互和材质，避免只做生硬名词替换。
- 用户指定平台时再读取对应平台文件；未指定时交付通用自然语言 Prompt。

## 自检

- 输出模式是否与用户请求一致？
- 模板是否只有一个占位符，且没有分析、负向内容或平台参数？
- 具体主体、品牌或文案是否被完整保留？
- 参考图中未请求迁移的身份元素是否已剥离，而风格信息仍足够具体？
