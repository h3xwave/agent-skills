# 检索策略参考

## 零、检索时间范围（全局约束）

所有检索查询中涉及的年份必须基于**系统提供的当前日期**动态计算，严禁硬编码任何具体年份。

设当前年份为 `Y`（从系统 `currentDate` 中提取当前日期，取其年份部分），则：

| 检索场景 | 年份范围 | 示例写法 |
|---------|---------|-------------------|
| 学术论文/开源动态（近期） | `Y-1` 至 `Y` | `{Y-1} {Y}` |
| 学术论文/技术趋势（扩展） | `Y-2` 至 `Y` | `{Y-2} {Y-1} {Y}` |
| 专利检索（需更宽窗口） | `Y-3` 至 `Y` | `{Y-3} {Y-2} {Y-1} {Y}` |
| 公开风险排查 | `Y-3` 至 `Y` | `{Y-3} {Y-2} {Y-1} {Y}` |

**搜索查询构建示例：**
```
# 正确 — 使用动态年份变量
multimodal database patent {Y-1} {Y} cross-modal retrieval

# 错误 — 把往年年份写死在检索式里（例如直接写具体四位年份）
```

## 一、关键词构建方法

### 1. 核心关键词提取
从交底书中提取以下三类关键词：
- **技术问题词**：描述要解决问题的关键术语
- **技术手段词**：描述解决方案的核心技术术语
- **技术效果词**：描述技术效果的关键术语

### 2. 关键词扩展策略
对每个核心关键词进行以下扩展：
- **同义词/近义词**：如"检测"→"识别/监测/探测"
- **上位词/下位词**：如"深度学习"→"机器学习"（上位）、"卷积神经网络"（下位）
- **中英文对照**：同时准备中文和英文关键词
- **缩写/全称**：如 CNN ↔ Convolutional Neural Network

### 3. 检索式构建
使用布尔逻辑组合关键词：
- `AND`：交集，缩小范围（如：A AND B）
- `OR`：并集，扩大范围（如：同义词之间用 OR）
- `NOT`：排除明显无关结果
- 通配符：`*` 匹配多字符，`?` 匹配单字符

示例检索式模板：
```
(技术手段词1 OR 同义词1) AND (技术手段词2 OR 同义词2) AND (应用领域词)
```

## 二、IPC/CPC 分类号参考

### 常用分类号查找方法
1. 通过 WIPO 的 IPC 分类表在线查询：https://www.wipo.int/en/web/classification-ipc
2. 通过已知相关专利的分类号反查同类专利
3. 常见技术领域对应的大类：
   - G06F：电数字数据处理（软件/算法）
   - G06N：基于特定计算模型的计算机系统（AI/机器学习）
   - G06V：图像或视频识别
   - H04L：数字信息传输（通信/网络）
   - G16H：医疗信息学
   - B25J：机器人/操纵装置

### 分类号使用建议
- 优先使用 IPC 小类（4位）+ 大组（斜杠前）进行初筛
- 结合关键词 + 分类号可显著提高检索精准度

## 四、检索结果数量调控

| 结果数量 | 操作 |
|---------|------|
| > 500 条 | 增加限定词（更具体的技术特征、更窄的分类号、限定申请日范围） |
| 50–500 条 | 适中，逐条筛选标题和摘要 |
| 20–50 条 | 较好，可逐条详细阅读 |
| < 20 条 | 扩展关键词（增加同义词、使用上位概念、去掉部分限定条件） |
| 0 条 | 检查拼写，更换关键词组合，尝试纯分类号检索 |
