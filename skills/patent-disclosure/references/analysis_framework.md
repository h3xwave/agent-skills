# 冲突对比分析框架

## 一、对比维度

对每篇高风险对比文件，从以下维度进行结构化对比：

| 对比维度 | 说明 |
|---------|------|
| 技术领域 | 是否属于相同或相近技术领域 |
| 技术问题 | 是否解决相同或类似的技术问题 |
| 技术手段 | 核心技术方案是否相同或等同 |
| 技术效果 | 是否达到相同或类似的技术效果 |
| 关键区别点 | 本发明与对比文件的本质差异 |

## 二、单篇对比表格式

对每篇高风险对比文件，输出以下格式的对比表：

```
┌──────────────┬──────────────────────┬──────────────────────┐
│ 对比维度      │ 本发明                │ 对比文件 CNXXXXXXXXX  │
├──────────────┼──────────────────────┼──────────────────────┤
│ 技术领域      │ [填写]               │ [填写]                │
│ 技术问题      │ [填写]               │ [填写]                │
│ 技术手段      │ [填写]               │ [填写]                │
│ 技术效果      │ [填写]               │ [填写]                │
│ 关键区别点    │ [本发明的差异化特征]    │ —                    │
├──────────────┼──────────────────────┼──────────────────────┤
│ 冲突程度      │ 高 / 中 / 低          │                      │
│ 影响的保护点   │ 保护点 X、Y           │                      │
└──────────────┴──────────────────────┴──────────────────────┘
```

## 三、冲突程度评级标准

### 高冲突
- 对比文件公开了与本发明**相同的技术手段**解决**相同的技术问题**
- 技术方案的核心步骤/结构高度重合（≥80%）
- 可能直接影响新颖性判断

### 中冲突
- 对比文件公开了**部分相同的技术手段**，或解决**相近的技术问题**
- 技术方案存在重合但有明显差异（40%–80%）
- 可能需要结合其他对比文件影响创造性判断

### 低冲突
- 对比文件仅在技术领域或背景技术上相关
- 技术方案的核心部分不同（重合 < 40%）
- 不太可能单独影响新颖性或创造性

## 四、过审率预估标准

### 分保护点过审率

根据每个保护点的冲突检索结果，按以下标准估算：

| 最高冲突等级 | 过审率范围 | 说明 |
|------------|----------|------|
| 无冲突 | 85%–95% | 未检索到相关对比文件，但不排除漏检风险 |
| 低冲突 | 70%–85% | 存在相关文件但核心方案不同，通过合理撰写可规避 |
| 中冲突 | 40%–70% | 需要突出区别特征，可能需要调整保护范围 |
| 高冲突 | 10%–40% | 存在高度相似的对比文件，需要显著缩小保护范围或调整方案 |

### 综合过审率

综合过审率基于各保护点的加权评估：

- **乐观情景**：所有区别特征均被审查员认可，取各保护点过审率的上限
- **基准情景**：部分区别特征被质疑，取中位值
- **悲观情景**：关键区别特征不被认可，取下限

### 影响过审率的关键因素

**有利因素：**
- 技术效果有明确的实验数据支撑
- 解决了现有技术未曾关注的技术问题
- 技术手段的组合方式具有非显而易见性
- 所属领域近期专利申请量少（审查宽松）

**不利因素：**
- 核心特征已被单篇对比文件公开
- 技术方案属于本领域常规技术手段的简单组合
- 缺少量化的技术效果数据
- 所属领域为热门申请领域（审查严格）

## 五、提高过审率的策略建议模板

根据检索结果，从以下角度给出针对性建议：

1. **缩小保护范围**：将独立权利要求限定至对比文件未公开的具体实施方式
2. **强化区别特征**：在技术方案中突出与对比文件的关键差异
3. **补充技术效果数据**：提供对比实验，量化本发明的优势
4. **增加从属权利要求层次**：形成梯度保护，即使独立权利要求被驳回仍有退路
5. **调整权利要求类型**：如方法权利要求冲突大，考虑增加装置/系统权利要求
