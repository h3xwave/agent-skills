# 专利交底书常用术语与表达参考

## 一、常用引导句式

### 技术领域

| 场景 | 模板 |
|------|------|
| 通用 | 本发明涉及[领域]技术领域，尤其涉及[具体方向]。 |
| 软件 | 本发明涉及人工智能技术领域，特别涉及基于[技术]的[应用]方法及系统。 |
| 硬件 | 本发明涉及[机械/电子/材料]领域，具体涉及一种[用途]的[装置/结构]。 |

### 背景技术过渡句

- "然而，上述方案存在以下不足："
- "但现有技术在[方面]仍存在以下问题："
- "现有[技术/产品/方法]在实际应用中存在如下缺陷："

### 发明目的句式

- "本发明的目的在于提供一种[名称]，以至少解决现有技术中存在的……问题之一。"
- "针对上述技术问题，本发明提出一种[名称]，旨在[目标效果]。"

### 技术方案连接词

- 包括/包含（开放式，推荐）
- 由…组成（封闭式，不推荐用于独立权利要求）
- 其特征在于（独立权利要求前序部分与特征部分的分界、从属权利要求引用部分之后均使用）
- 其中 / 所述
- 在一个实施例中 / 在一些实施方式中

### 有益效果句式

- "本发明通过[技术手段]，实现了[技术效果]，从而[最终价值]。"
- "与现有技术相比，本发明的有益效果在于："
- "本发明至少具有以下技术优势："

## 二、第5节"关键点和欲保护点"撰写参考（供代理人转化为权利要求）

### 独立权利要求三段式结构

```
[前序部分]一种[名称]，
[特征部分]其特征在于，包括：
  [技术特征1]；
  [技术特征2]；
  [技术特征3]。
```

### 从属权利要求结构

```
根据权利要求[N]所述的[名称]，其特征在于，[进一步限定内容]。
```

### 方法权利要求结构

```
一种[名称]的方法，包括以下步骤：
  S1、[步骤1描述]；
  S2、[步骤2描述]；
  S3、[步骤3描述]。
```

### 系统/装置权利要求结构

```
一种[名称]系统，包括：
  [模块1名称]，用于[功能1]；
  [模块2名称]，用于[功能2]；
  [模块3名称]，与所述[模块1名称]连接，用于[功能3]。
```

## 三、不同技术类型发明名称规范

| 类型 | 推荐命名格式 | 示例 |
|------|------------|------|
| 方法类 | 一种[技术特征]+[用途]的方法 | 一种基于深度学习的图像去噪方法 |
| 装置类 | 一种[技术特征]+[用途]的装置/设备 | 一种高效散热的锂电池组件 |
| 系统类 | 一种[技术特征]+[用途]的系统 | 一种分布式实时数据处理系统 |
| 结构类 | 一种[特征描述]+[名称] | 一种多层复合隔热结构 |

命名注意事项：
- ❌ 不要用：最佳、最优、超级、新型（夸大宣传）
- ❌ 不要用：发明人名字、商品名
- ✅ 使用技术特征描述，直接反映发明核心

## 四、常见错误与纠正

| 错误描述 | 正确表达 |
|---------|---------|
| "我们的系统比其他系统好" | "与现有技术相比，本发明具有……优势" |
| "用了一个新的算法" | "采用基于[原理]的[算法名称]" |
| "效果很好" | "将[指标]提升了XX%，降低了……" |
| "任何材料都可以" | "所述材料选自[A、B、C]中的一种或多种" |
| "大概是这样连接的" | "所述[部件A]通过[连接方式]与所述[部件B]连接" |

## 五、敏感内容处理

### 涉密信息
- 如发明涉及公司核心商业秘密，提醒发明人：交底书提交前需确认保密审查流程
- 建议在"补充说明"中标注"含有公司保密技术信息，请勿对外公开"

### 在先公开问题
- 如发明已在论文、会议、产品上公开，需注明公开日期
- 中国专利法设有 6 个月的优惠期（针对特定情形），需提醒发明人

### 职务发明认定
- 标准：在本职工作范围内完成，或主要使用了公司资源
- 发明人有权在交底书中说明个人贡献比例

## 六、专利类型选择建议

| 类型 | 适用场景 | 保护期 | 审查时间 |
|------|---------|--------|---------|
| 发明专利 | 技术方案具有创造性，核心竞争力 | 20年 | 2-3年 |
| 实用新型 | 产品结构改进，需快速授权 | 10年 | 6-12个月 |
| 外观设计 | 产品外观、形状、图案 | 15年 | 6-12个月 |

> 注：同一产品类技术方案可在同日分别申请发明专利和实用新型，并在申请时声明，俗称"一案两报"；发明授权时须放弃实用新型，只能保留一项。方法类方案不能申请实用新型，不适用该做法。
