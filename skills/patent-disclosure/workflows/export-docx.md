# Word 导出流程

## 用途与边界

用户要求生成 `.docx` 时使用。导出属于快速模式，不要求联网检索、公开风险确认或风险预判；只转换用户已确认的 Markdown 内容。

## 输入约束

- 输入文件必须是非空 UTF-8 Markdown，第一条有效内容应是 `# 发明名称` 或纯文本发明名称，且正文至少包含一个章节标题。
- 支持纯数字章节标题（如 `1 `、`2.1 `）和 Markdown 标题。用户模板优先，不要求固定五章。
- 不支持 Markdown 表格；脚本会在生成前将其作为输入错误报告，避免把表格线静默写成普通正文。
- Mermaid 代码块可以保留，但脚本不渲染 Mermaid，只替换为“请手动插图”的占位文字。

## 本地 CLI 环境

1. 确认本地 Python 3 环境已安装 `python-docx`。
2. 直接运行脚本，无需先读取源码：

```text
python <path-to-skill>/scripts/generate_docx.py <输入.md> <输出.docx>
```

3. 脚本保持两个位置参数，并使用以下退出码：
   - `0`：生成成功。
   - `2`：用法或输入校验错误。
   - `1`：依赖、读取或写入等运行错误。
4. 生成后确认输出文件存在，并抽查标题、章节、分页和 Mermaid 占位文字。

## 受限环境

- 如果环境支持 Python 执行，优先直接调用 `scripts/generate_docx.py`。
- 缺少 `python-docx` 时说明依赖缺失；是否安装依赖遵循当前环境权限，不擅自扩大操作范围。
- 无法执行脚本时返回已校验的完整 Markdown，并给出本地转换命令。

## 样式与实现

| 元素 | 规范 |
|---|---|
| 页面 | A4，页边距上/下 0.75 英寸、左/右 1 英寸，1.5 倍行距，页脚页码 |
| 发明名称 | 居中，14pt，楷体_GB2312，加粗 |
| 正文/章节标题 | 楷体_GB2312，12pt，黑色；章节标题加粗 |
| 全文西文 | Times New Roman |
| 图片处理 | Mermaid 仅保留占位文字；其他图片需用户在 Word 中插入或由后续工具处理 |

## 完成检查

- 输入校验已通过，输出 `.docx` 存在。
- 页面尺寸为 A4，用户章节结构未被强制改为默认五章。
- 已明确 Mermaid 不渲染及图片处理限制。
- 未因导出任务触发联网检索或预判报告。
