# 专利交底书 Skill

`patent-disclosure` 用于把发明想法、研发材料或已有交底书整理成供企业知识产权部门或专利代理机构复核的技术材料。默认面向中国专利实践并输出中文 Markdown。

英文文档见 [README.md](README.md)。本 Skill 不替代专利代理师、律师或正式法律意见，也不承诺授权结果或生成可直接提交的正式申请文件。

## 能力范围

- 从宽泛研发方向中梳理候选保护点。
- 通过少量、自适应问题补齐发明信息。
- 检索并整理专利、论文、标准、产品与公开实现等现有技术。
- 按保护点给出定性风险、证据覆盖度、置信度和未知项。
- 起草、补强或润色专利交底书。
- 整理可供代理人转化的关键保护点和权利要求素材。
- 将符合输入约束的 Markdown 交底书导出为 `.docx`。

以下任务不属于其交付边界：代替代理人形成可直接提交的权利要求书、作出确定法律结论、承诺授权率、代用户提交申请。

## 两种工作模式

### 完整模式

适合只有发明想法或希望同时获得现有技术评估的用户。流程依次完成公开风险确认、自适应访谈、现有技术风险评估、用户确认和交底书起草。

专利现有技术检索不限定在最近几年，而是围绕保护点、分类号、引用链和相关日期上限覆盖历史资料。报告使用高/中/低风险，不输出固定“过审率”。

### 快速模式

适合用户已经提供较完整材料，或只要求整理、润色、格式调整、保护点提炼、Word 导出。Skill 会提示未检索或未评估的局限，但不会强制联网、预判报告或固定轮次访谈。

## 典型请求

```text
我有一个传感器结构创新，想走完整流程形成给代理人的交底材料。
```

```text
这是已有研发说明，请先快速整理成五章交底书草稿，暂时不要联网。
```

```text
检索这个机械连接结构的历史现有技术，并按保护点给出定性风险。
```

```text
把已经定稿的交底书 Markdown 导出为 Word，不要重新检索。
```

## Word 导出

导出脚本保持以下调用方式：

```powershell
py -3 skills/patent-disclosure/scripts/generate_docx.py input.md output.docx
```

输出使用 A4 页面。脚本不会渲染 Mermaid；代码块会转换为手动插图提示，本地图片也需在 Word 中后处理。输入格式和其他限制以 Skill 内的导出流程为准。

## 安装与验证

```powershell
py -3 tests/validate_repository.py --skill patent-disclosure
.\scripts\install-skill.ps1 -Name patent-disclosure -DryRun
.\scripts\install-skill.ps1 -Name patent-disclosure
```

仓库中的 `behavior_cases.json` 是静态契约检查；真实输出与触发边界分别记录在 `evals.json` 和 `trigger_evals.json`，不随 Skill 安装。

## 时效与保密

联网检索只在方向挖掘、现有技术评估或用户明确要求时执行。未公开的项目代号、内部参数、客户名称、源代码和完整方案不得直接作为外部查询词；应先抽象为技术问题、技术手段和技术效果。

检索结论受数据库覆盖、检索截止日期和未公开资料限制，应交由专业代理人复核。
